密码模式

Java24K 2021-12-01

写在前面：Oauth2系列配套视频地址 https://t.hk.uy/bdr8

# 一.概念

密码模式（Resource Owner Password Credentials Grant）中，用户向客户端提供自己的用户名和密码。客户端使用这些信息，向"服务商提供商"索要授权。

# 二.适用场景

这种模式会直接将用户密码暴露给应用程序，因此应谨慎使用。一般说来，只有信任度极高的客户才应授权使用该模式，如官方移动应用、操作系统或高权限程序。

# 三.示意图与时序图

授权验证流程示意图

A-B-C

（A）用户向客户端提供用户名和密码。

（B）客户端将用户名和密码发给认证服务器，向后者请求令牌。

（C）认证服务器确认无误后，向客户端提供访问令牌。

授权验证流程时序图


# 四.流程与 参数详解

B步骤中，客户端发出的HTTP请求，包含以下参数：

参数名称 描述
grant_type 表示授权类型，此处的值固定为"password"，必选项。
username 表示用户名，必选项。
password 表示用户的密码，必选项。
scope 可选。请求可能会有一个或多个scope值。授权服务器会把客户端请求的范围(scope)展示给用户看。

C步骤中，认证服务器向客户端发送访问令牌

HTTP/1.1 200 OK Content-Type: application/json;charset=UTF-8

Cache-Control: no-store

Pragma: no-cache { "access_token":"2YotnFZFEjr1zCsicMWpAA", "token_type":"example", "expires_in":3600, "refresh_token":"tGzv3JOkF0XG5Qx2TlKWIA", "example_parameter":"example_value"}

# 五.实例演示解析

1. 获得token

http://localhost:8080/oauth/token?grant_type=password&username=lisi&password=123456

2. 拿着token访问资源

http://localhost:8080/oauth2/user-name?access_token=71689e9e-e738-4e3d-87b9-943fc2d54478 (opens new window)
