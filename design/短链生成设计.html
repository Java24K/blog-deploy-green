短链生成设计

Java24K 2022-07-02

# 一.短链如何生成

前提：6 位 62 进制数可表示 568 亿的数，应付长链转换绰绰有余。

# 长网址Hash方式

  1. 长网址hash得到一个十进制hash值A。
  2. 查询hash值A是否存在数据库，存在则长网址后拼接随机数再进行hash。
  3. 将十进制hash值A转化为62进制值B。
  4. 拼接62进制值B拼到路径后面得到短链 https://域名/B值。
  5. 将长网址、短链长、长网址10进制hash值 存入数据库。

# 分布式ID方式

  1. 生成分布式ID与长网址一一对应。
  2. 将分布式ID转化为62进制值B。
  3. 拼接62进制值B拼到路径后面得到短链 https://域名/B值。
  4. 将长网址、短链、分布式ID存入数据库。

二者相比，长网址hash会冲突需要解决，分布式ID方式没有冲突。推荐分布式ID方式。


# 二.短链如何查询

短网址是非10进制的数字,对计算机来说就是一串字符,直接存到数据库不好做范围查询.

  1. 针对分布式ID方式，分布式ID就是短链值的十进制。在查询的时候短网址中的B值62进制转换成10进制，根据分布式ID查询。
  2. 针对长网址hash方式，长网址10进制hash值是数字。在查询的时候短网址B值62进制转换成10进制，根据长网址10进制值查询。

# 三.浏览器如何访问

# 如何访问长连接

根据短链查询到长链后，服务端http响应状态码为301，进行了重定向。 举例：当我们访问链接 https://t.hk.uy/bdGE 的时候用google的http插件我们能看到

# 301与302区别

301 302
类型 永久重定向 临时重定向
描述 301表示旧地址A的资源已经被永久地移除了（这个资源不可访问了） 302表示旧地址A的资源还在（仍然可以访问），这个重定向只是临时地从旧地址A跳转到地址B
搜索引擎 抓取新内容的同时也将旧的网址交换为重定向之后的网址 搜索引擎会抓取新的内容而保存旧的网址

# 301和302共同点

就是说浏览器在拿到服务器返回的这个状态码后会自动跳转到一个新的URL地址，这个地址可以从响应的Location首部中获取（用户看到的效果就是他输入的地址A瞬间变成了另一个地址B）。


# 四.短链注释事项

  1. 短链生成验证，要有验证措施，比如点击转换后输入验证码啊等，不然会被人家把服务器打爆。
  2. 短链生成限制，限制IP的单日请求总数，超过阈值则直接拒绝服务。
  3. 短链生成缓存，缓存提高查询效率，如用redis缓存 长网址->短网址 的映射，仅存储一天以内的数据，用LRU机制进行淘汰。
