Redis排行榜设计

Java24K 2022-09-03

# 一.业务场景

排行榜业务场景比较多如 微信步数排行、新浪热门话题排行、王者荣耀战绩排行等。

  1. 一般应用于大量数据的排行，如top10。
  2. 对实时性要求相对较高，如当前top10.

# 二.排行榜实现方式

  1. 通过数据库实现，在小数据量的场景中数据库order by可以实现。
  2. 通过Redis实现，大数据量的排行用Redis的sorted set 数据结构。

# 三.Redis的Sorted Set介绍

# 存储结构

zset底层的存储结构包括ziplist或skiplist，在同时满足以下两个条件的时候使用ziplist，其他时候使用skiplist，两个条件如下：

  • 有序集合保存的元素数量小于128个
  • 有序集合保存的所有元素的长度小于64字节

# SkipList

数据量比较大时Redis 的 zset 是一个复合结构由 hash 和 skiplist组成。



  1. 一方面它需要一个 hash 结构来存储 member 和 score 的对应关系，键为成员，值为分值。

  2. 另一方面「跳跃列表」提供按照 score 来排序成员member的功能。

我们就以微信步数排行榜微例子


# 如何存储

2021-02-27这一天的排行榜用skiplist存储


如果score步数相同会按照member的字典顺序排序


# 涉及命令

  1. 添加 member 语法：zadd key score member [score member ...]
  2. 增加 member 的 score 语法：zincrby key increment member
  3. 获取 member 排名 : 从低到高排名 zrank key member，从高到低排名zrevrank key member。
  4. 返回指定排名范围内的 member：zrange/zrevrange key start end [withscores] ，当带上withsocres这个参数之后，会返回对应 member 的 score。

# 排行榜个人视角

微信步数排行榜每个人看到的都不相同

  • 我们只需要在key上加上用户名作为区分即可,如 sprt:ranking:why:20210227

# 四.排行榜问题扩展


# 最近七天排行榜

求why同学微信步数最近七天排行榜，用并集

  • 使用命令 zunionstore destination numkeys key [key ...] [weights weight [weight ...]] [aggregate sum|min|max] 获取并集

求why同学最近七天里都上传微信步数的同学的排行榜，用交集

  • 使用命令 zinterstore destination numkeys key [key ...] [weights weight [weight ...]] [aggregate sum|min|max] 获取交集

# 亿级用户排行榜

像王者荣耀这样的上亿用户做排行榜该怎么做？

  1. 使用分治的思想，王者排行分为8个段位，分别是青铜、白银、黄金、铂金、钻石、星耀、最强王者和荣耀王者。

  1. 这样在计算的时候why哥的排名就是 x+y+why哥在星耀的排名。当然8个桶太少，我们还可以分，8个段位每个段位又有不同的级别，每个级别都可以是一个桶。
