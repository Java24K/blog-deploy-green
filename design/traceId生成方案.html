traceId生成方案

Java24K 2022-07-03

# 一.Http调用traceId生成使用方案

# 方案描述

  1. 加入统一filter，设置traceId到MDC即可，MDC无需清理。
  2. 下次请求时规则：若request中存在X-Request-Id则获取作为traceId，若没有，则生成UUID作为traceId。

# 具体实现

  1. 添加过滤器，查询request是否存在traceId，存在使用，不存在通过UUID生成。
  • 定义过滤器
public class TraceFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest httpServletRequest, HttpServletResponse httpServletResponse, FilterChain filterChain) throws ServletException, IOException {
        boolean bInsertMDC = insertMDC(httpServletRequest);
        try {
            filterChain.doFilter(httpServletRequest, httpServletResponse);
        } finally {
            MDC.remove("traceId");
        }
    }

    private boolean insertMDC(HttpServletRequest httpServletRequest) {
        String requestTraceId = httpServletRequest.getHeader("traceId");
        if(StringUtils.isNotEmpty(requestTraceId)){
            MDC.put("traceId", requestTraceId);
        }else{
            UUID uuid = UUID.randomUUID();
            String traceId = uuid.toString().replace("-", "");
            MDC.put("traceId", traceId);
        }
        return true;
    }
}
  • 添加过滤器
@Configuration
public class FilterConfig {

    @Bean
    public FilterRegistrationBean traceFilterRegistration() {
        FilterRegistrationBean registration = new FilterRegistrationBean();
        registration.setFilter(new TraceFilter());
        registration.addUrlPatterns("/*");
        registration.addInitParameter("paramName", "paramValue");
        registration.setName("TraceFilter");
        registration.setOrder(0);
        return registration;
    }
}
  1. 在调用端发起http请求时，将traceId写到header中

# 二.Dubbo调用traceId生成方案

# 消费端更改

  1. 消费端新增一个Filter类
@Activate(group = {Constants.CONSUMER, Constants.PROVIDER} , order = -9999)
public class GlobalTraceFilter implements Filter {

    @Override
    public Result invoke(Invoker<?> invoker, Invocation invocation) throws RpcException {
        String traceId = invocation.getAttachment("traceId");
        if(!StringUtils.isBlank(traceId)) {
            RpcContext.getContext().setAttachment("traceId",traceId);
        }else { // 第一次发起调用
            RpcContext.getContext().setAttachment("traceId", UUID.randomUUID().toString());
        }
        return invoker.invoke(invocation);
    }
}
  1. 新增配置文件src\main\resources\META-INF\dubbo\com.alibaba.dubbo.rpc.Filter并编辑内容
dubboRpcFilter=com.alibaba.dubbo.rpc.filter.TraceFilter
  1. 修改consumer.xml，对生产者应用此过滤器
<!-- 提供方调用过程缺省拦截器，将拦截所有service -->
	<dubbo:provider timeout="30000" filter="dubboRpcFilter"/>

# 生产端更改

  1. 生产端自定义拦截器中获取traceId，放入MDC。
public class ContextFilter implements Filter
{
    @Override
    public Result invoke(Invoker<?> invoker, Invocation invocation)
        throws RpcException
    {
        String traceId = RpcContext.getContext().getAttachment("traceId");
        if (StrKit.notBlank(traceId)) {
        	MDC.put("traceId", traceId);
        }
    	
    	Result result = invoker.invoke(invocation);
        return result;
    }


}
  1. 新增配置文件src\main\resources\META-INF\dubbo\com.alibaba.dubbo.rpc.Filter并编辑内容
dubboContextFilter=com.alibaba.dubbo.rpc.interceptor.ContextFilter
  1. 修改provider.xml，对生产者应用此拦截器
<!-- 提供方调用过程缺省拦截器，将拦截所有service -->
	<dubbo:provider filter="dubboContextFilter"/>

备注：无论是生产端还是消费端都要修改log4j文件，输出格式中加上traceId变量（%X{traceId}），这样才能在日志中看到traceId。
