Java注解

Java24K 2021-12-01

# 一.定义注解

  • 概念：一个注解准确意义上来说，只不过是一种特殊的注释而已，如果没有解析它的代码，它可能连注释都不如。通过 @interface 关键字进行定义。

  • 举例：

// 创建一个名字为 TestAnnotaion 的注解
public @interface TestAnnotation { }

# 二.应用注解

  • 举例：
// test类加上@TestAnnotation，使用上面定义的注解 
@TestAnnotation public class Test { }

# 三.解析注解

  • 解析注解方式有两种：一种是编译期直接的扫描，一种是运行期反射（根据类反射得到方法上注解）。可以通过AOP切面对注解进行统一操作添加处理逻辑

# 四.JAVA注解与属性

# JAVA元注解

  • 定义：所谓元注解其实就是可以注解到别的注解上的注解,被注解的注解称之为组合注解,组合注解具备其上元注解的功能。java有四个元注解，如下所示。
  1. @Target：注解的作用目标，其中ANNOTATION_TYPE 用于注解类型声明即注解注解。

@Target(ElementType.TYPE)   //接口、类、枚举、注解

@Target(ElementType.FIELD) //字段、枚举的常量

@Target(ElementType.METHOD) //方法

@Target(ElementType.PARAMETER) //方法参数

@Target(ElementType.CONSTRUCTOR)  //构造函数

@Target(ElementType.LOCAL_VARIABLE)//局部变量

@Target(ElementType.ANNOTATION_TYPE)//注解

@Target(ElementType.PACKAGE) ///包

  1. @Retention：注解的保留位置

@Retention(RetentionPolicy.SOURCE)   //注解仅存在于源码中，在class字节码文件中不包含

@Retention(RetentionPolicy.CLASS)     // 默认的保留策略，注解会在class字节码文件中存在，但运行时无法获得，

@Retention(RetentionPolicy.RUNTIME)  // 注解会在class字节码文件中存在，在运行时可以通过反射获取到

  1. @Documented：注解是否应当被包含在 JavaDoc 文档中

  2. @Inherited：是否允许子类继承该注解

# JAVA内置注解举例

  1. @Override 伪代码,表示重写(当然不写也可以)。

  2. @Deprecated 用于表示某个元素（类、方法等）已过时。

  3. @SuppressWarnings 用于抑制编译器产生警告信息。

  4. @FunctionalInterface 表示是函数式接口，接口只能有一个抽象方法，多个则会报错。

# JAVA注解的属性

  1. 注解的成员变量在注解的定义中以“无形参的方法”形式来声明

  2. 注解中属性可以有默认值，默认值需要用 default 关键值指定。

# JAVA自动定义注解

// 定义注解
@Target(ElementType.TYPE)
@Retention(RetentionPolicy.RUNTIME)
public @interface TestAnnotation {     
    int id();     
    String msg(); 
}
// 使用注解
@TestAnnotation(id=3, msg="hello annotation")
public class Test {}
